# Selector Adaptation Guide

Этот документ описывает лучшие практики и подходы к работе с селекторами для различных web3 кошельков, методы их отладки и обновления.

## Как адаптировать селекторы под разные версии MetaMask
MetaMask часто обновляет DOM, что может приводить к поломкам жестких (xpath/структурных) селекторов.
1. **Используйте `data-testid`**: Если разработчики MetaMask добавляют тестовые атрибуты (`data-testid`), всегда используйте их (`[data-testid="page-container-footer-next"]`).
2. **Текстовые селекторы как fallback**: Используйте поиск по тексту (`hasText: "Next"`, `text="Confirm"`) осторожно, обращайте внимание на локализацию. Если кошелек открывается на другом языке, текстовый селектор упадет.
3. **Избегайте жестких структурных привязок**: Исключите селекторы вроде `div > div > span:nth-child(2)`. Ищите элемент по роли, например: `button:has-text("Connect")`.
4. **Учитывайте А/В тесты**: Существуют версии, где экран подтверждения может быть разделен на два шага. Ожидайте оба варианта через `Promise.race` или метод `.or()` в Playwright.

## Как адаптировать селекторы под разные версии Rabby
Rabby Wallet имеет сложный UI с большим количеством всплывающих окон и модалок внутри самого расширения.
1. **Тени и Shadow DOM**: Убедитесь, что селекторы способны проникать в возможные web-components, если Rabby начнет их использовать.
2. **Работа с фокусом**: Rabby часто требует явного сброса фокуса или клика в рабочую область перед вводом пароля.
3. **Button variants**: Rabby использует много стилизованных кнопок. Лучше всего привязываться к уникальным иконкам или классам, непосредственно отвечающим за логику (например, `button.rabby-btn-primary`).
4. **Всплывающие предупреждения**: Rabby любит показывать секьюрити-предупреждения. Ваш скрипт должен проверять наличие кнопки "Ignore" или "Proceed anyway" перед тем, как нажать основную кнопку "Confirm".

## Как дебажить popup pages
Отладка страниц расширений (popup) отличается от обычных веб-страниц.
1. **Использование Playwright Inspector**: Запускайте выполнение с `PWDEBUG=1`.
2. **Получение URL расширения**: В Playwright вы можете найти background page или service worker расширения и перейти на его URL (например, `chrome-extension://<id>/popup.html`). Это позволит открыть popup как обычную вкладку на весь экран, что упрощает инспектирование через DevTools.
3. **Паузы в коде**: Вставляйте `await page.pause()` прямо перед моментом, где селектор падает, чтобы исследовать DOM вручную.
4. **Скриншоты и HTML-дампы**: Настройте автоматический дамп DOM и скриншот, если элемент не найден за отведенный таймаут.

## Как логировать unknown wallet states
Когда кошелек переходит в состояние, не описанное в ваших сценариях (например, новый экран онбординга или критическая ошибка ноды):
1. Выбрасывайте `NeedsReviewError` с кодом `wallet_unknown_state`.
2. Обязательно сохраняйте скриншот и дамп HTML дерева страницы перед закрытием браузера. Артефакты должны быть привязаны к `runId`.
3. В логах фиксируйте URL страницы кошелька и тексты нескольких ключевых элементов на экране, чтобы понять контекст по логам без проверки скриншотов.

## Как безопасно обновлять detector rules
1. **Не удаляйте старые селекторы сразу**: Добавляйте новые селекторы как дополнительное условие. Если структура изменилась, старый селектор просто не сработает со временем, а новый подхватит работу.
2. **Оценка Confidence Score**: В `WalletDetector` используйте несколько признаков для уверенности. Например, наличие глобального объекта `window.ethereum` дает 0.5 к уверенности, а наличие специфичного DOM элемента внутри провайдера или URL – еще 0.5.
3. **Изолированное тестирование**: Перед коммитом проверьте новые правила на сохраненных HTML-дампах от старых версий (мокирование).

## Как писать fallback rules
Playwright предоставляет удобные механизмы для создания fallback цепочек.
1. Использование комбинаций `locator.or()`:
    ```typescript
    const confirmBtn = page.locator('button[data-testid="confirm-btn"]')
        .or(page.locator('button:has-text("Approve")'))
        .or(page.locator('.primary-button.confirm'));
    await confirmBtn.click();
    ```
2. Использование массивов с коротким таймаутом:
    ```typescript
    const selectors = ['#next', '.btn-next', 'button >> text="Next"'];
    for (const sel of selectors) {
      if (await page.locator(sel).isVisible({ timeout: 1000 })) {
         await page.locator(sel).click();
         break;
      }
    }
    ```

## Как тестировать detector локально
1. Поднимите чистый профиль браузера программно (через скрипт `scripts/local-test.ts`).
2. Поместите в профиль нужные версии распакованных расширений.
3. Напишите простой CLI интерфейс, который будет вызывать методы детектирования (`WalletManager.detect()`) и пытаться разблокировать кошелек, передав тестовый пароль.
4. Настройте локальный сервер, возвращающий фиктивные dapp страницы, чтобы скрипт вызывал появление popup, и прогоните workflow подтверждения транзакции.
